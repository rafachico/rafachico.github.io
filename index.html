Projetos em Produção

Problema

Acesso à legislação de zoneamento e licenciamento empresarial era fragmentado, exigia conhecimento técnico-jurídico e consumia horas de pesquisa em portais públicos.

Solução

Fine-tuning supervisionado de um modelo Llama com QLoRA + Unsloth sobre o corpus jurídico municipal completo, exposto via interface de linguagem natural. O modelo responde em segundos consultas que antes levavam horas.

Stack Técnica

LlamaQLoRAUnsloth HuggingFacePythonFastAPI

Impacto

Democratização do acesso à informação técnica municipal — de horas para segundos. Interface de linguagem natural elimina a barreira de conhecimento jurídico.

Duas Variantes

Modelo Python Fine-Tuned — geração e análise de código Python: 97,95% LLM-as-Judge (GPT-3.5: 94,29%) · CodeBERT-F1 0,9131
Modelo SQL Fine-Tuned — text-to-SQL em dialect corporativo: 71,33% LLM-as-Judge (GPT-3.5 Turbo: 67,52%) · CodeBERT-F1 0,9459

Dataset Sintético

74.222 exemplos de treino gerados por pipeline automatizado: extração de contexto de código-base real → geração de pares instrução/resposta via LLM → filtragem de qualidade → dataset estruturado para fine-tuning supervisionado.

Treinamento

QLoRA 4-bit com LoRA rank-16 em NVIDIA A100 40 GB. Fev–Abr 2024. Avaliação tripla: LLM-as-Judge, CodeBERT-F1 e execução funcional de código.

Disponibilidade & Impacto

Modelos públicos em Hugging Face, compatíveis com API OpenAI via vLLM para deploy corporativo. Quantização 4-bit reduz custo de memória e acelera inferência em ambientes de produção.

Stack Técnica

Code Llama 7BQLoRAPEFT HuggingFacevLLMCodeBERT A100 40GBPython

Módulos

Gerador de Workflows — constrói fluxos LLM com catálogo de conectores, detecção de ambiguidade em prompts e continuidade de sessão entre agentes
Engine de Guardrails — NER via Transformers+spaCy, Regex, Moderação, detecção de Jailbreak e Prompt Injection, SDK de Blocking/Masking de PII em tempo real
Observabilidade — stack Loki + MongoDB + Fluentd, rastreamento distribuído de chamadas entre agentes e sistema de alertas
Web Search — roteador multi-provider: Tavily · Exa · DuckDuckGo · SerpAPI
MCP Server — protocolo de contexto para integração com agentes externos

Diferencial

Auditoria completa de todas as decisões tomadas por agentes, detecção de PII em tempo real com mascaramento antes de qualquer chamada a LLM externo, conformidade a PLD e LGPD com trilhas de log imutáveis.

Guardrails em Ação

Cenário real de proteção: solicitações com padrões de CPF e dados sensíveis são interceptadas por NER + Regex antes do LLM. O pipeline aplica mascaramento automático e segue com resposta segura, preservando rastreabilidade para auditoria posterior.

Contexto

Uma instituição financeira precisava formalizar o ciclo de vida de modelos de IA em produção — de aprovação até monitoramento contínuo — em conformidade com PLD, LGPD e diretrizes do BACEN sobre uso responsável de algoritmos em decisões de crédito e risco.

Entregáveis

· Mapeamento de riscos e criação de Model Cards para todos os modelos em produção
· RBAC granular com políticas de acesso segmentadas por time e nível de risco
· Pipeline de monitoramento de concept drift e data drift com alertas automáticos
· Processo de validação de viés algorítmico pré-deploy com relatório de equidade
· Trilhas de auditoria imutáveis para decisões automatizadas de alto impacto

Stack Técnica

PythonMLFlowRBAC Bias DetectionModel Cards PLDLGPDPostgreSQL

Impacto

Redução de risco regulatório, visibilidade completa sobre o ciclo de vida dos modelos e processos formais de revisão humana para decisões automatizadas de alto impacto ao cliente.

Arquitetura (3 etapas)

1. Detecção facial com MTCNN · 2. Geração de embeddings com Inception ResNet / ResNet50 · 3. Matching por Cosine Similarity contra galeria de identidades.

Performance

VGGFace2: 97.77% Precision · 97.22% Recall. Deploy otimizado para CUDA no NVIDIA Jetson com gestão de dependências cross-arch (Python 3.6 · Torch 1.10).

Datasets

VGGFace2 (3,3M+ faces · 9.131 identidades) · CASIA WebFace (494k imagens · 10.575 identidades)

Problema

Identificar falhas silenciosas na caldeira do sistema de aquecimento de água central antes que causem interrupção de serviço — sem depender de manutenção reativa.

Arquitetura

Sonda DS18B20 → ESP8266 com firmware ESPHome (MQTT over Wi-Fi) → Home Assistant (automações e alertas) hospedado em Raspberry Pi 3B local.

Funcionalidades

Dashboard de temperatura em tempo real · Alertas automáticos por anomalia de leitura · Histórico de séries temporais · Detecção de falha por threshold configurável.

Stack Técnica

ESP8266ESPHome Home AssistantRaspberry Pi 3B MQTTDS18B20

Como Funciona

Software de captura (host) amostra as bordas do frame em tempo real, computa a cor média por região e envia os valores RGB via serial USB ao Arduino, que controla cada LED da fita WS2812B individualmente via protocolo endereçável.

Resultado

Latência perceptível < 50 ms · Suporte a múltiplos perfis de layout por posição da fita (superior, lateral, inferior) · Extensível para múltiplos segmentos independentes.

Stack Técnica

ArduinoWS2812BFastLED Serial USBC++

Infraestrutura

VPS Linux com cluster Kubernetes single-node — ambiente deliberadamente não gerenciado para estudar os primitivos K8s (deployments, services, ingress, secrets) sem abstração de cloud provider.

Aplicação — Notificações E-commerce

n8n hospedado no cluster para orquestrar notificações de eventos do e-commerce: confirmação de pedido, atualização de rastreio e alertas de status — integrando webhooks do e-commerce com múltiplos canais de notificação.

Stack Técnica

Abordagem

Pipeline com embeddings contextuais e fine-tuning de modelos transformers para classificação robusta de sentimento em domínio real.

Aplicações

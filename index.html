Projetos em Produção

Clique em qualquer card para expandir o tech breakdown completo.

Problema

Acesso à legislação de zoneamento e licenciamento empresarial era fragmentado, exigia conhecimento técnico-jurídico e consumia horas de pesquisa em portais públicos.

Solução

Fine-tuning supervisionado de um modelo Llama com QLoRA + Unsloth sobre o corpus jurídico municipal completo, exposto via interface de linguagem natural. O modelo responde em segundos consultas que antes levavam horas.

Stack Técnica

LlamaQLoRAUnsloth HuggingFacePythonFastAPI

Impacto

Democratização do acesso à informação técnica municipal — de horas para segundos. Interface de linguagem natural elimina a barreira de conhecimento jurídico.

Módulos

Orquestração — gerencia o ciclo de vida e comunicação entre agentes de IA · Segurança — guardrails com detecção e mascaramento de PII em tempo real · Fluxo de Dados — controle e rastreabilidade do pipeline entre serviços de IA.

Diferencial

Auditoria completa de todas as decisões tomadas por agentes, com conformidade a regulamentações financeiras (PLD, Bacen, ANS) e trilhas de log imutáveis.

Stack Técnica

PythonFastAPIn8n LLM GuardrailsPII DetectionAudit Logs JavaScript

Contexto

Uma seguradora precisava adotar IA em processos críticos de underwriting, precificação e análise de sinistros de forma responsável, rastreável e em conformidade com LGPD e circulares da SUSEP.

Entregáveis

· Mapeamento de riscos e criação de Model Cards para todos os algoritmos em produção
· RBAC granular com políticas de acesso segmentadas por time e nível de risco
· Pipeline de monitoramento de concept drift com alertas automáticos
· Processo de validação de viés algorítmico pré-deploy
· Trilhas de auditoria para decisões automatizadas de alto impacto

Stack Técnica

PythonFastAPIMLFlow RBACBias DetectionModel Cards LGPDSUSEPPostgreSQL

Impacto

Redução de risco regulatório, visibilidade completa sobre o ciclo de vida dos modelos e processos formais de revisão humana para decisões de alto impacto ao segurado.

Arquitetura (3 etapas)

1. Detecção facial com MTCNN · 2. Geração de embeddings com Inception ResNet / ResNet50 · 3. Matching por Cosine Similarity contra galeria de identidades.

Performance

VGGFace2: 97.77% Precision · 97.22% Recall. Deploy otimizado para CUDA no NVIDIA Jetson com gestão de dependências cross-arch (Python 3.6 · Torch 1.10).

Datasets

VGGFace2 (3,3M+ faces · 9.131 identidades) · CASIA WebFace (494k imagens · 10.575 identidades)

Stack Técnica

MTCNNInception ResNetResNet50 PyTorchCUDANVIDIA Jetson FastAPIOpenCV

Abordagem

Extração de features com ResNet50, normalização por histogram matching (scikit-image) e busca KNN no espaço de embeddings com StandardScaler. Múltiplas métricas: FSIM, SSIM, RMSE, UIQ.

API

POST /api/img_fraud_detect  POST /api/images_upload

Stack Técnica

ResNet50KNNscikit-image FastAPIGradioOpenCV NumPyscikit-learn

Arquitetura (4 containers)

Redis (session store) → OAuth2 Proxy (Google Auth + JWT) → Nginx (reverse proxy) → MLFlow 1.28 (SQLite + GCS artifacts).

Segurança

Controle de acesso por domínio via Google Cloud. Cookie refresh de 2h. Header forwarding: X-User X-Email X-Access-Token.

Frameworks suportados

SKLearnTensorFlowPyTorch XGBoostYOLOv5

Stack Técnica

Docker ComposeNginxOAuth2 Proxy RedisGCSGCP

Estágios

1. SAST — security scanning · 2. Unit Test — Pytest Python 3.9 · 3. Validate — Dockerfile lint · 4. Build — Docker → GCR · 5. Deploy — Cloud Run · 6. Healthcheck — POST /health

Diferencial

Docker-in-Docker (DinD) para build isolado, autenticação via service account GCP e verificação automatizada de saúde de frontend e backend após cada deploy.
